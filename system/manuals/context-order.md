# 컨텍스트 최적화 순서

## 정보 전달 순서 (고정)

AI에게 정보를 전달할 때 아래 순서를 반드시 따른다.
순서가 잘못되면 AI 성능이 떨어지고 토큰이 낭비된다.

```
1. 역할 정의 (누구인가)
2. 규칙/제약 (하지 말아야 할 것)
3. 컨텍스트 (현재 상황)
4. 태스크 (해야 할 것)
5. 출력 형식 (어떻게 답할 것)
```

## 상세 설명

### 1단계: 역할 정의

```
당신은 TypeScript 전문 프론트엔드 개발자입니다.
```

- 짧고 명확하게
- 전문 분야 특정

### 2단계: 규칙/제약

```
- any 타입 사용 금지
- console.log 금지
- 500줄 이상 파일 금지
```

- coding-rules.md 참조
- 프로젝트별 추가 규칙

### 3단계: 컨텍스트

```
현재 프로젝트: 쇼핑몰 대시보드
기술 스택: Next.js + TypeScript + Tailwind
현재 파일: src/components/ProductList.tsx (변경 대상)
관련 타입: src/types/product.ts
```

- 관련 파일만 최소한으로
- 변경 대상 파일 명시

### 4단계: 태스크

```
ProductList 컴포넌트에 페이지네이션 기능을 추가하세요.
- 한 페이지에 20개 항목
- 이전/다음 버튼
- 현재 페이지 표시
```

- 구체적 요구사항
- 체크리스트 형태

### 5단계: 출력 형식

```
변경된 코드만 출력하세요.
파일 경로를 포함하세요.
```

- 필요한 출력만 요청
- 불필요한 설명 요청 안 함

## 프로젝트 지침 파일 구조

각 프로젝트의 INSTRUCTIONS.md는 위 순서를 따른다:

```markdown
# 프로젝트명

## 역할

이 프로젝트의 AI 개발자 역할 정의

## 규칙

코딩 규칙 + 금지 사항

## 현재 상태

진행 중인 작업, 주요 파일 구조

## 태스크 큐

현재 해야 할 작업 목록
```
