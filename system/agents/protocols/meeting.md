# 에이전트 회의 프로토콜

## 회의 유형

### 1. 설계 회의 (Design Review)

- 참석: architect, developer, reviewer
- 트리거: 새 기능 또는 구조 변경 시
- 결과물: 설계 결정서

### 2. 코드 리뷰 회의 (Code Review)

- 참석: developer, reviewer, tester
- 트리거: PR 생성 또는 구현 완료 시
- 결과물: 리뷰 결과 + 수정 사항

### 3. 회고 회의 (Retrospective)

- 참석: 전체 에이전트
- 트리거: 마일스톤 완료 시
- 결과물: 개선 사항 목록

## 회의 흐름

```
1. 의제 설정 (architect 또는 트리거 에이전트)
2. 각 에이전트 의견 제시
3. 충돌 발생 시 → 투표
4. 합의 → 결정 기록
5. 회의록 자동 생성
```

## 충돌 해결 규칙

- architect 의견 가중치: 2x
- 동률 시: architect가 최종 결정
- 보안 이슈: reviewer 거부권 보유

## 회의록 형식

```markdown
# 회의록 - [유형] - [날짜]

## 참석 에이전트

## 의제

## 논의 내용

## 결정 사항

## 액션 아이템
```
