# 회고 프로토콜

## 트리거

- 마일스톤 완료
- 스프린트 종료
- 심각한 버그 발견 후
- 수동 실행

## 회고 흐름

```
1. 데이터 수집
   - 완료된 태스크 목록
   - 커밋 히스토리
   - 에러/실패 로그
   - 토큰 사용량

2. 분석
   - 잘된 점 (Keep)
   - 개선할 점 (Problem)
   - 시도할 점 (Try)

3. 지침 갱신 제안
   - 매뉴얼 수정 제안
   - 새 규칙 추가 제안
   - 불필요한 규칙 제거 제안

4. 승인 → 반영
   - 사용자 승인 필수
   - 승인된 변경만 반영
```

## 회고 보고서 형식

```markdown
# 회고 보고서 - [프로젝트] - [날짜]

## 요약

완료 태스크: N개 | 에러 발생: N건 | 토큰 사용: N

## Keep (유지)

## Problem (문제)

## Try (시도)

## 지침 갱신 제안

| 대상 파일 | 변경 유형 | 내용 |
| --------- | --------- | ---- |

## 승인 상태

- [ ] 사용자 승인 대기
```
